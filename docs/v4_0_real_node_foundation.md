# V4.0 Real Node Foundation

## 1. Stage Goal

V4.0 establishes the real node foundation. It turns the transaction entry path from experiment-input replay into real signed transactions admitted by independent node-level mempools.

Target segment:

```text
real trace / client -> signed tx -> node submit -> per-node mempool
```

## 2. Allowed Scope

V4.0 may add:

```text
executor/cmd/mbe-node/
executor/cmd/mbe-client/
executor/cmd/mbe-supervisor/
executor/realism/tx/
executor/realism/account/
executor/realism/mempool/
executor/realism/node/
executor/realism/config/
executor/realism/metrics/
```

V4.0 may implement:

- signed transaction model;
- deterministic local signing/verifying for emulator use;
- account and nonce model;
- duplicate tx detection;
- per-node mempool;
- mempool admission policy;
- mempool capacity, TTL, priority, and wait metrics;
- real trace import into signed tx records;
- minimal long-running node lifecycle skeleton;
- minimal client submit command;
- V4.0 logs and summary.

## 3. Forbidden Scope

V4.0 must not implement or claim:

- real PBFT;
- real P2P gossip;
- block proposal;
- block commit;
- persistent state db;
- cross-shard commit;
- frontend Realism Mode;
- production blockchain security;
- BlockEmulator superiority.

## 4. Required Transaction Fields

A V4.0 transaction must have:

```text
tx_id
sender
receiver
nonce
value
state_keys
payload
signature
timestamp
source_kind
```

Optional fields:

```text
gas_limit
gas_price
chain_id
shard_hint
trace_source_id
```

## 5. Required Mempool Behavior

Each node must own an independent mempool.

Required methods:

```text
Admit(tx)
Reject(tx, reason)
Contains(tx_id)
PendingCount()
Select(max_count)
Expire(now)
Snapshot()
```

Required admission checks:

- valid tx id;
- valid signature;
- known sender;
- nonce order;
- duplicate tx id;
- capacity;
- TTL.

## 6. Required Artifacts

V4.0 should output:

```text
signed_tx_import_log.csv
node_admission_log.csv
node_mempool_log.csv
nonce_validation_log.csv
signature_validation_log.csv
v4_node_foundation_summary.json
```

## 7. Acceptance Criteria

A V4.0 smoke run passes only if:

1. multiple `mbe-node` instances can start with distinct node IDs;
2. each node has its own mempool object and log;
3. `mbe-client` can submit signed transactions;
4. invalid signatures are rejected;
5. duplicate tx ids are rejected;
6. nonce violations are rejected or held according to policy;
7. valid transactions enter the target node mempool;
8. logs are generated by node admission behavior, not static fixture rows;
9. V3 runtime remains runnable;
10. docs clearly state that PBFT/P2P/state/cross-shard are not yet implemented in V4.0.

## 8. Suggested Smoke Commands

Exact commands can be adjusted after code implementation, but the intended shape is:

```powershell
cd executor
go run ./cmd/mbe-node --node-id n0 --shard-id s0 --config ../../configs/v4/local_4node.json

go run ./cmd/mbe-client --target n0 --tx-count 100 --seed 1 --signed true

go test ./...
```

## 9. Stage Truth Label

```text
v4_real_node_foundation
```

This label means only the transaction admission and per-node mempool path is real. It does not imply real P2P, PBFT, state commit, or cross-shard commit.

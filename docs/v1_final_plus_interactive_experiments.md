# V1-final-plus interactive experiments

V1-final-plus is an acceptance-layer integration for interactive V1 experiment
configuration. It does not add a new core mechanism, it is not V1.9, and it
does not enter V2/V3 scope.

## What it adds

The web UI and FastAPI backend now expose a small interactive V1 run path:

- choose a workload or trace source;
- set small replay parameters such as `tx_count`, `seed`, `hot_tx_ratio`,
  `conflict_injection_ratio`, `commutative_update_ratio`, `access_set_size`,
  and `multi_hotspot_count`;
- choose an ablation preset or custom toggles;
- run a local replay into `.cache/v1_custom_runs/latest/`;
- inspect base, routing, dual-track, and hot-update aggregation metrics;
- download `summary.csv`, `latency.csv`, `runtime.log`, `trace_meta.json`,
  `used_config.yaml`, `used_config.json`, and `report.md` when available.

## Data source labels

V1-final-plus keeps data-source semantics explicit:

- **Synthetic replay** generates a local V1 trace using `asset_hotspot_v1` or
  `reward_burst`, then replays it with the Go executor. This is not real chain
  execution.
- **Existing trace replay** replays a `trace.jsonl.gz` already present inside
  the project workspace. It does not generate workload or launch any chain.
- **Chain-backed replay** replays `.cache/fabric_smoke/latest/trace.jsonl.gz`
  generated by the V1.4 Fabric smoke runner. The web UI only replays the trace.
- **Real Fabric smoke** remains a CLI/WSL operation. The web UI intentionally
  does not start Docker, Fabric, `network.sh`, deploy chaincode, or invoke
  peers.

Recommended Fabric smoke command:

```bash
cd ~/MBE
source .venv/bin/activate
python scripts/v1_fabric_smoke.py --strict --channel mbechannel --out .cache/fabric_smoke/latest
```

## Ablation presets

The interactive run supports the V1.8 baseline/ablation presets:

- `baseline_hash_only`: hash routing, dual-track disabled, hot aggregation
  disabled.
- `co_access_only`: co-access routing, dual-track disabled, hot aggregation
  disabled.
- `co_access_dual_track`: co-access routing, dual-track enabled, hot
  aggregation disabled.
- `full_v1`: co-access routing, dual-track enabled, hot aggregation enabled.
- `custom`: manually choose routing, dual-track, and aggregation toggles.

These presets configure already-implemented V1.5/V1.6/V1.7 mechanisms. They do
not change the algorithms.

## Boundaries

V1-final-plus remains single-chain replay and acceptance tooling. It does not
implement:

- formal dual-chain or multi-chain execution;
- cross-chain protocols;
- MetaFlow;
- committee bridge;
- Pending Pool;
- production Fabric;
- multi-server Fabric.

Generated custom-run artifacts are written under `.cache/v1_custom_runs/`, which
is ignored by Git.

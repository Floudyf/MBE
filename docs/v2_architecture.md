# V2 Architecture

## 1. Architecture Overview

```text
Frontend Experiment Console
        |
        v
Experiment Composer 2.0
        |
        v
Config Validator ---- Plugin Registry
        |
        v
Job Manager ---- Artifact Manager
        |
        v
Trace Source Layer
  | synthetic
  | existing trace
  | Fabric chain-backed trace
  | public-chain imported trace
        |
        v
Replay Layer
  | single-chain V1 executor
  | dual-chain replay engine
  | cross-chain protocol replay
        |
        v
Metrics / Sweep / Report
```

## 2. Relationship to V1

V2 reuses V1. V1 remains the runnable `single_chain` plugin set.

V2 does not rewrite the V1 executor. V2 does not remove V1 workload, trace, routing, dual-track, aggregation, Fabric smoke, or V1-final-plus UI.

V1 capabilities become the initial plugin base:

```text
topology: single_chain
workloads: asset_hotspot_v1, reward_burst
trace_sources: synthetic, existing_trace, fabric_chain_backed_trace
routing: hash, co_access
execution: serial, dual_track
commit: normal, hot_update_aggregation
reports: v1_8_sweep, custom_run_report
```

## 3. Plugin Registry

V2 defines these plugin types:

```text
topology plugin
workload plugin
trace source plugin
routing plugin
execution plugin
commit plugin
chain backend plugin
cross-chain protocol plugin
report plugin
```

Each plugin has at least:

```text
id
label
version
status: runnable | planned | experimental | invalid
stage
inputs
outputs
dependencies
non_goals
truth_label
```

## 4. Experiment Composer 2.0

Composer 2.0 generates an experiment config from user selections. It fills default values, calls the validator, returns `runnable` / `planned` / `invalid`, and explains why a configuration cannot run.

Status definitions:

```text
runnable: current code can run the experiment
planned: planned for a future stage, not runnable
experimental: runnable with explicit limitations
invalid: illegal combination
```

## 5. Trace Source Layer

Trace source types must be kept distinct:

```text
synthetic trace: simulated workload with controlled parameters, not real on-chain execution
Fabric chain-backed trace: trace generated by real Fabric smoke, web platform only replays it
existing trace: user-provided or previously generated trace
public-chain imported trace: imported public-chain data, default semantic_unknown
```

Public-chain imported traces are externally realistic but usually lack reliable access-set, dependency, and commutative-delta semantics unless additional tracing and contract-specific decoding are provided.

## 6. Replay Layer

V2 separates replay into three layers:

```text
single-chain replay: reuse the V1 executor
dual-chain replay: implemented in V2.5
cross-chain protocol replay: implemented in V2.6
```

V2.5/V2.6 remain virtual-time replay. They are not a production cross-chain bridge.

## 7. MetaFlow Relationship

MetaFlow is not implemented in V2.1.

V2.5 and V2.6 provide the experimental substrate needed by MetaFlow. MetaFlow may later be added as a protocol plugin. MetaFlow should not be mixed into plugin registry or trace schema before the substrate is ready.

## 8. Data Truth Labels

```text
synthetic_replay: generated workload plus executor replay; not real chain execution
chain_backed_replay: replay using a trace generated by a chain-backed source
fabric_smoke: CLI/WSL Fabric test-network run that deploys chaincode and invokes transactions
public_chain_imported_trace: imported public-chain data with externally real origin and default unknown semantics
production_deployment_planned: production-like deployment is reserved for V3 planning
```
